# sudoku-ruby.rb 概要

## 目的

9×9の数独パズルを、人間が行う論理的推論を段階的に適用することで自動的に解くプログラム。
各セルに「候補数字の集合」を持たせ、数独のルール（行・列・3×3ブロック内に1〜9が1つずつ）に従って候補を絞り込み、全セルの候補が1つになれば解決とみなす。

## クラス構成

| クラス | 役割 |
|--------|------|
| `Cell` | セル1マス。候補数字の集合 `@possibles` を保持。候補が1つになると「固定」とみなす |
| `CellList` | 行・列・ブロックを表す9セルのリスト。制約伝播のメソッド群を持つ |
| `Table` | 9×9盤面全体を管理し、`CellList` に処理を委譲してパズルを解く |
| `TableLoader` | 9文字×9行の文字列からパズルの初期配置を読み込む |
| `Util` | `Set[1..9] - set` を返す補集合ヘルパー |

## 使用している手法

解法は `Table#solve` が4段階の推論を収束するまで繰り返すことで実現されている。

### Level 0 — 基本制約伝播（`check`）

`CellList#check` が行・列・ブロックそれぞれについて2種類の推論を行う。

1. **固定セルの波及**: 候補が1つに絞られたセルの数字を、同グループ内の他セルの候補から除去する。
2. **隠れシングル（Hidden Single）**: あるグループ内で特定の数字を置けるセルが1つだけの場合、そのセルをその数字に固定する。

この2ステップをオープンセル数が変化しなくなるまで繰り返す。

### Level 1 — 隠れシングル（`find_and_fix_uniq_possible`）

`@impossibles`（候補にない数字の集合）を使い、グループ内でその数字が不可能なセルの数を数えることで唯一の配置先を特定する。Level 0 の `check` とは独立した実装で補完的に働く。

### Level 2 — ネイキッドサブセット（`find_and_fix_with_fixed_chair`）

N個のセルがまったく同じN個の候補を持つ場合（例：2セルが `{3, 7}` のみ）、そのN個の数字はそのN個のセルが独占するとみなし、グループ内の他セルの候補から除去する。

### Level 3 — ブロック・ライン相互作用（`fix_with_cross_block`）

ブロック内で特定の数字が1つの行（または列）にしか現れない場合、その数字はそのブロック内で確定するため、同じ行（列）のブロック外のセルから除去できる（Pointing Pairs/Triples）。

### 収束ループ

`solve` は上記を順に適用し、オープンセル数が前回から変化しなくなるか0になるまで繰り返す。

## 改善が可能な点

### 1. バックトラッキングの未実装

現在の実装は論理推論のみで解く。論理だけでは解けない最高難度のパズル（「最難問」「悪魔の数独」など）には対応できない。`Cell` に `take_snapshot` / `restore_snapshot` が用意されているが `solve` では使われていない。仮置き＋矛盾検出によるバックトラッキングを追加すれば、あらゆるパズルに対応できる。

### 2. `CellList.@@cache` のグローバル状態汚染

`@@cache` はクラス変数のためプロセス全体で共有される。`Table.new` しても前回のキャッシュは残るため、複数パズルを同一プロセス内で連続して解こうとすると誤った `CellList` が返る。`@@cache` のクリアメソッドを追加するか、キャッシュを `Table` インスタンスに持たせる設計に変える必要がある。

### 3. Level 3 でのキャッシュ肥大化

`fix_with_cross_block` が呼ばれるたびに `new_with_subtract` で生成される一時的な `CellList` が `@@cache` に蓄積し続ける。これらは使い捨てなのでキャッシュに登録すべきでない。

### 4. `fix_with_cross_block` のセル単位ループによる非効率

現在はオープンセルごとにブロック→行・列の演算を繰り返しているが、同じブロック内のセルで同じ計算を何度も行う。ブロック単位のループに変更すれば計算量を最大9分の1に削減できる。

### 5. Level 0 と Level 1 の役割の重複

`CellList#check`（Level 0）と `found_uniq_possible_numbers`（Level 1）はいずれも「グループ内で唯一配置可能なセルを見つける」処理を行っている。実装方法が異なるためゼロではないが、役割を整理して統合できる余地がある。

### 6. `count_of_open_cell` の非効率

`open_cells` が毎回配列を生成してから `length` を返している。カウントだけが必要な場合は配列生成を避けてカウンタを返すほうが効率的。

### 7. 未使用コードの整理

- `Cell#take_snapshot` / `restore_snapshot`: バックトラッキング用として用意されているが未使用。
- `CellList#open_fixed_list`: 定義されているが呼び出し箇所がない。
- `CellList#new_with_cross`: 定義されているが使われていない。

### 8. `TableLoader` の設計

`initialize` が空のクラスであり、実質 `TableLoader.new.read(...)` という呼び出しパターンのみ。クラスメソッド `TableLoader.read(str_matrix)` にすれば冗長な `new` を省ける。

### 9. エラーメッセージのタイポ

- `"contadictory"` → `"contradictory"`
- `"impassible"` → `"impossible"`
